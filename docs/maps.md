# Карты в Pathformer

## Сколько карт создаётся

Генерируется **много карт** — датасет содержит тысячи уникальных карт, каждая с уникальным `seed`.
Каждая карта хранится в HDF5-файле (`/maps/{map_id}/`), датасет делится на train/val/test по картам:
- 80% карт — обучение
- 10% — валидация
- 10% — тест

На каждой карте строится несколько траекторий с разными стартом, целью и классом судна.
Таким образом модель учится обобщаться на **незнакомые карты**, а не запоминать конкретные пути.

## Структура одной карты

Одна карта — это три 2D-массива `(H, W)`, созданные вместе через `generate_map()`:

| Слой | Тип | Диапазон | Описание |
|------|-----|----------|----------|
| `land_mask` | float32 | 0.0 / 1.0 | Бинарная маска суши |
| `current_intensity` | float32 | [0, 3] м/с | Скорость течения (0 на суше) |
| `current_direction` | float32 | [0, 2π) рад | Направление течения |

### Генерация land_mask
Гауссов шум → сглаживание (σ=20) → порог на 85-й перцентиль → ~15% суши.

### Генерация current_intensity
Гауссов шум → сглаживание (σ=30) → нормировка в [0, 3] м/с → обнуление на суше.

### Генерация current_direction
Curl noise: шум → сглаживание (σ=25) → градиент потенциала → перпендикуляр к градиенту.
Curl noise даёт физически корректное поле без источников и стоков.

## Как нейросеть использует слои карты

Нейросеть **не предсказывает по слоям по отдельности** — она объединяет их в единый тензор
и обрабатывает совместно через CNN.

### Шаг 1 — Препроцессинг (preprocessor.py)

Три сырых слоя преобразуются в 3-канальный тензор `(3, H, W)`:

```
Канал 0: u = intensity * cos(direction) / max_current   — восточная составляющая скорости
Канал 1: v = intensity * sin(direction) / max_current   — северная составляющая скорости
Канал 2: safety_field = 1 - clip(dist_to_land / 20, 0, 1)  — близость к суше
```

`land_mask` и `current_direction` исчезают как отдельные каналы — их информация
закодирована в `u`, `v` и `safety_field`.

### Шаг 2 — CNN-энкодер (encoder.py)

Все три канала поступают одновременно в CNN:

```
(B, 3, H, W)
  → ConvBlock(3→32,   stride=2)   # H/2
  → ConvBlock(32→64,  stride=2)   # H/4
  → ConvBlock(64→128, stride=2)   # H/8
  → ConvBlock(128→128, stride=2)  # H/16
  → ConvBlock(128→128, stride=1)  # H/16 (уточнение)
  → (B, 128, H/16, W/16)
```

CNN сам учится, какие комбинации каналов важны для навигации.
Например, высокая скорость течения в направлении пути — это попутное течение (хорошо),
против пути — встречное (плохо). CNN улавливает такие паттерны автоматически.

### Шаг 3 — Токенизация и cross-attention (planner.py, decoder.py)

```
(B, 128, H/16, W/16)
  → flatten → (B, N_tokens, 128)
  → + 2D positional encoding
  → cross-attention с waypoints декодера
```

Для карты 256×256: H/16 = 16, итого 16×16 = 256 токенов карты.
Декодер на каждом шаге генерации пути "смотрит" на токены карты через cross-attention,
фокусируясь на релевантных участках (напр., узких проходах между островами).

Карта кодируется **один раз** в начале инференса — токены переиспользуются на всех шагах.

## Карта стоимостей (вспомогательная)

Помимо нейросетевого пайплайна, при генерации обучающих данных строится cost map для A*:

```
cost = 1.0 + 1.0*(I/3.0) + 10.0*(I > vessel_max_current) + inf*(land_mask)
```

- Базовая стоимость воды: 1.0
- Сильное течение: +штраф пропорционально интенсивности
- Сверхлимитное течение: +10.0
- Суша: бесконечность

Эта карта используется A* для построения опорных траекторий датасета,
а также в `collision_repair.py` для исправления коллизий нейросетевого пути.
