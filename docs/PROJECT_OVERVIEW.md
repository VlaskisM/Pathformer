# Pathformer — что это и как устроено

## Простыми словами

Это **нейросеть, которая планирует маршрут для беспилотного морского судна**. На входе — карта моря (где суша, где течения, насколько они сильные и куда текут) + точка старта и точка цели. На выходе — список точек (waypoints), по которым судну нужно пройти от старта к цели, не врезаясь в сушу и учитывая течения.

Классический способ решать такую задачу — алгоритм **RRT\*** (или A\*) — работает точно, но медленно (секунды). Pathformer обучается на решениях этих классических алгоритмов и потом выдаёт почти такой же хороший маршрут **за доли секунды**. Это и есть весь смысл проекта.

**Цели:** маршрут не длиннее чем на 20% больше оптимального, успешно доходит до цели в 85%+ случаев, считается меньше 200 мс на GPU.

## Архитектура нейросети

```
Карта (3 канала, 256×256)
        │
        ▼
┌──────────────────┐
│   CNN Encoder    │   ← сжимает картинку в компактное представление
│   (5 блоков)     │
└──────────────────┘
        │
        ▼
   256 "токенов карты" (как слова для трансформера)
        │
        ▼                   ← сюда же подаём: старт, цель, класс судна
┌──────────────────┐
│ Transformer      │   ← генерирует путь шаг за шагом
│ Decoder          │      (авторегрессивно: каждая следующая
│ (4 слоя)         │       точка зависит от предыдущих)
└──────────────────┘
        │
        ▼
   Waypoints: (x1,y1) → (x2,y2) → … → (goal)
```

**Что такое 3 канала карты:**
- `u`, `v` — компоненты вектора течения (куда и насколько сильно течёт)
- `land_mask` — 1 если суша, 0 если вода

**Как именно работает decoder:**
Он не выдаёт весь путь сразу, а предсказывает **смещение** на каждом шаге (`Δx`, `Δy`), потом применяет его, и повторяет. Остановка — когда дошёл до цели (или превысил лимит шагов).

## Структура репозитория

```
pathformer/
├── src/pathformer/              ← сама нейросеть
│   ├── config.py                ← все гиперпараметры (d_model, n_layers, lr, ...)
│   ├── model/
│   │   ├── encoder.py           ← CNN, сжимающий карту
│   │   ├── decoder.py           ← Transformer, генерирующий путь
│   │   ├── planner.py           ← связывает encoder + decoder
│   │   └── components.py        ← строительные блоки (attention, conv, positional encoding)
│   ├── data/                    ← загрузка датасета для обучения
│   ├── training/trainer.py      ← обучение (train loop)
│   ├── inference.py             ← выполнение модели в проде
│   ├── metrics.py               ← метрики качества
│   └── path_utils.py            ← сглаживание пути сплайнами
│
├── synthetic_generator/         ← генератор искусственных данных
│   └── generate_map/            ← создаёт карты процедурно
│                                  (шум → суша, шум → течения, A* → оптимальный путь)
│
├── scripts/                     ← CLI-скрипты
│   ├── train.py                 ← обучение
│   ├── evaluate.py              ← измерить качество на тестовых данных
│   └── visualize.py             ← нарисовать путь картинкой
│
├── tests/                       ← pytest-тесты
├── best.pt                      ← готовая обученная модель (checkpoint)
└── pyproject.toml               ← зависимости
```

## Что за чем запускать

### Шаг 1. Установка
Нужен Python 3.12+ и PyTorch 2.5+.
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu124
pip install -e ".[dev]"
```

### Шаг 2. Данные (одно из двух)

**Вариант А: сгенерировать синтетические данные** (если нет реальных карт):
```bash
cd synthetic_generator
python generate_dataset.py --output ../data/dataset.h5 --num_maps 50 --pairs_per_map 100
```
Что произойдёт:
1. Создастся 50 случайных карт (суша + течения из шума Перлина)
2. На каждой карте выбирается 100 пар старт/цель
3. Для каждой пары A\* считает оптимальный путь
4. Всё сохраняется в HDF5-файл

**Вариант Б: использовать свои карты** — сложить файлы в `data/` по формату из README.

### Шаг 3. Обучение
```bash
python scripts/train.py --data_dir data/ --device cuda
```
Учится, пока не достигнет нужного качества. Лучший чекпоинт → `checkpoints/best.pt`.

### Шаг 4. Проверка качества
```bash
python scripts/evaluate.py --checkpoint checkpoints/best.pt --data_dir data/
```
Выведет: success rate, cost ratio, Frechet distance.

### Шаг 5. Использование в коде
```python
from pathformer.model.planner import USVPlanner
# загрузить модель из best.pt
path, success = model.plan(x_map=карта, start=старт, goal=цель, vessel_class=класс_судна)
# path — готовые waypoints
```

## Классы судов

Разные лодки по-разному борются с течениями:

| Класс | Макс. течение | Поведение |
|-------|---------------|-----------|
| `light_usv` | 0.5 м/с | Избегает сильных течений — длинные обходы |
| `medium_usv` | 1.0 м/с | Умеренно |
| `heavy_usv` | 2.0 м/с | Прёт напролом через любые течения |

Модель получает класс судна на вход и учитывает его при планировании.

## Как генерируется синтетическая карта (простыми словами)

1. **Суша**: берём случайный шум, размазываем по Гауссу, отсекаем порогом так, чтобы ~15% площади стало сушей
2. **Сила течений**: аналогично — шум + размытие, значения 0–3 м/с
3. **Направление течений**: curl-шум (гарантированно вихревое поле без источников/стоков — физически правдоподобно)
4. **Карта стоимости**: каждой клетке присваивается цена прохождения (суша = ∞, сильное течение = дорого)
5. **Путь**: A\* ищет самый дешёвый путь от старта до цели
6. **Ресемплинг**: путь переразбивается на равные отрезки (10–200 точек)

## Где что искать, если…

- **Хочу поменять размер модели** → `src/pathformer/config.py`
- **Хочу добавить новый класс судна** → `synthetic_generator/generate_map/config.py` (параметры) + `src/pathformer/data/` (как фичи подаются в модель)
- **Модель плохо учится** → логи обучения в `training/trainer.py`, проверить lr, warmup, batch_size
- **Хочу увидеть путь глазами** → `python scripts/visualize.py --data_dir data/ --checkpoint best.pt --index 0`
- **Нужен сервис/API вокруг** → это уже отдельный проект (map-service), pathformer — это ядро модели
